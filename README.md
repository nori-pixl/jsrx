# jsrx

jsrx は、バックトラック（探索の手戻り）が発生しない O(N) 時間 で動作する、超高速かつ安全な自作の有限オートマトン（NFA）型正規表現ライブラリです。

従来のJavaScript標準の RegExp で発生する恐れのある ReDoS（正規表現サービス拒否攻撃）を完全に回避し、悪意のある長い文字列に対しても絶対にフリーズしません。さらに、直感的で流暢なインターフェース（Fluent Interface）と、SQL URL解析などの実用的なプリセットを備えています。

## 特徴

- ReDoS完全耐性: Thompsonの構成法とTag-based VMを採用し、常に文字数に比例した最速タイムで処理を完了します。
- 直感的なAPI: rx(全体文字列, パターン) という、直感的にマッチ部分を「取る」だけのシンプルな設計。
- 1ファイル・ゼロ依存: 外部ライブラリを一切使わず、たった1つのファイルで完結しているため軽量でバックアップも容易です。
- 強力なプリセット: rx.num や、URLからSQLクエリを自動で引っこ抜く rx.sql を標準搭載。

## インストール・利用方法

GitHub Pages（CDN）経由で、HTMLの script タグから1行で直接読み込んで利用できます。

```html
<script type="module">
  // あなたのリポジトリから直接インポート
  import { rx } from 'https://github.io';

  // 1. 通常のパターンマッチ（数字の抽出）
  const number = rx.num("エラーコード: 503");
  console.log(number); // "503"

  // 2. カッコ()を使った特定部分の超高速抽出
  const patternMatch = rx("ユーザー名: john_doe", "ユーザー名: (.*)");
  console.log(patternMatch); // "john_doe"

  // 3. SQLの接続URL自動解析
  const myUrl = "mysql://admin:pass@localhost:3306/db?exec=(SELECT * FROM users WHERE id = 1)";
  const query = rx.sql(myUrl);
  console.log(query); // "SELECT * FROM users WHERE id = 1"
</script>
```

※ ブラウザの自動翻訳機能が有効な場合、JavaScriptのコードが破壊されてエラーになることがあります。正常に動作しない場合は、ブラウザの翻訳を「オフ（元に戻す）」に設定してください。

## サポートしている正規表現の種類

現在、以下の構文をバックトラックなしの純粋な有限オートマトンにコンパイルして実行できます。

| 構文 | 意味 |
| :--- | :--- |
| . | 改行を除く任意の1文字 |
| * | 0回以上の繰り返し（ループ） |
| + | 1回以上の繰り返し |
| \| | 選択（左ルートまたは右ルートのOR検索） |
| \d | 数字クラス（0から9） |
| (...) | グループ化（通過した文字の位置を記録して抽出） |

## アーキテクチャ（中身の仕組み）

jsrx は、一般的な正規表現エンジン（NFAのバックトラック型）とは異なり、Googleの RE2 などと同じ 「バーチャルマシン（VM）型オートマトン」 の仕組みをJavaScript上で表現しています。

1. コンパイルフェーズ: 渡された正規表現を左から解析し、Thompsonの構成法を用いて「状態遷移グラフ（State）」を組み立てます。
2. 実行フェーズ: 入力されたテキストを1文字ずつ読み進めながら、可能性のあるすべての状態を Set で同時に並行移動（走査）させます。
3. キャプチャ: グラフ内に仕込まれた START / END タグを通過したインデックスを記録し、マッチ終了時に一瞬で文字列を切り出します。
